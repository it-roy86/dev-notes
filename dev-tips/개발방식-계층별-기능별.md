# 개발 방식 비교 — 계층별 개발 vs 기능별 개발

> 작성일: 2026-04-18
> 참고 프로젝트: meal-management

---

## 두 가지 개발 방식

### 방식 1 — 계층별 개발 (Layer-by-Layer)

전체 구조를 한 계층씩 완성해 나가는 방식이에요.

Entity (DB 테이블) → Repository (DB 접근) → Service (비즈니스 로직) → Controller (API) → 화면 (Vue.js)

장점
- 전체 구조가 탄탄하게 잡힘
- 팀원들과 역할 분담하기 좋음
- 나중에 유지보수가 쉬움

단점
- 결과물이 늦게 보임
- 초반에 동기부여가 떨어질 수 있음

적합한 상황
- 팀 프로젝트
- 규모가 큰 프로젝트
- 장기적인 유지보수가 필요한 프로젝트

---

### 방식 2 — 기능별 개발 (Feature-by-Feature)

화면 하나를 완성하면서 필요한 기능을 하나씩 붙여가는 방식이에요.

화면 1 (일일 식사 입력) → Controller → Service → Repository → Entity
화면 2 (현황 조회) → Controller → Service → Repository → Entity
화면 3 (월별 정산) → Controller → Service → Repository → Entity

장점
- 결과물이 빨리 보임
- 동기부여가 유지됨
- 기능 하나가 완성될 때마다 성취감이 있음

단점
- 나중에 코드 구조가 복잡해질 수 있음
- 팀 프로젝트에서는 역할 분담이 어려울 수 있음

적합한 상황
- 혼자 개발하는 사이드 프로젝트
- 빠른 프로토타입이 필요할 때
- 작은 규모의 프로젝트

---

## 비교 표

| | 계층별 개발 | 기능별 개발 |
|--|--|--|
| 결과물 속도 | 느림 | 빠름 |
| 코드 구조 | 탄탄함 | 유연함 |
| 동기부여 | 초반에 낮음 | 꾸준히 높음 |
| 팀 협업 | 유리 | 불리 |
| 적합한 규모 | 중/대형 | 소형 |

---

## 핵심 정리

어떤 방식이 더 좋다는 정답은 없어요.
프로젝트 규모, 팀 구성, 개발 기간에 따라 상황에 맞는 방식을 선택하는 것이 중요해요!

혼자 개발 + 사이드 프로젝트 → 기능별 개발 추천
팀 프로젝트 + 장기 운영 → 계층별 개발 추천
